## 目录约定

| 类型 | 位置 |
|------|------|
| 训练产物 | `outputs/exp_xxx/` |
| 实验计划和汇总 | `experiments/` |
| 不再使用 | `experiments/exp_xxx*/` 这类实验产物目录 |

已将 `experiments/exp_021*`、`experiments/exp_022*`、`experiments/exp_023*` 的实验数据迁到：

- `outputs/exp_021/`
- `outputs/exp_022/`
- `outputs/exp_023/`

后续新增实验统一写入 `outputs/exp_xxx/`。

## 总原则

| 项 | 要求 |
|----|------|
| Base config | `configs/models/convnext_tiny.yaml` |
| 统一输入 | `--data.image_size 320 --training.batch_size 32` |
| 输出目录 | `outputs/exp_xxx`，必须同时传 `--experiment_id exp_xxx` 和 `--logging.experiment_name exp_xxx` |
| 记录指标 | macro F1、rainy F1、val loss、best epoch、训练时间 |
| 汇总文件 | 每完成一批后更新 `experiments/results.csv` 和 `experiments/leaderboard.md` |
| 不做事项 | 不再继续 ConvNeXt 256/384、大幅增强、MixUp、RandAugment、Rotation 20 |

命令模板：

```bash
python3 scripts/train.py \
  --config configs/models/convnext_tiny.yaml \
  --output_dir outputs \
  --experiment_id exp_xxx \
  --notes "实验说明" \
  -- \
  --logging.experiment_name exp_xxx \
  --data.image_size 320 \
  --training.batch_size 32 \
  ...
```

## 汇总判定

| 判定项 | 标准 |
|--------|------|
| 主模型选择 | 多 seed mean macro F1 领先 >= 0.002 |
| rainy 路线保留 | rainy F1 明显提升，macro F1 下降不超过 0.001 |
| weight decay 调整 | `wd=0.0005` 同时降低 val loss 并提高 macro F1 |
| EMA/SWA 晋级 | macro F1 提升 >= 0.001 且 val loss 下降 |
| ensemble 晋级 | macro F1 提升 >= 0.002 且 CPU 推理时间仍在预算内 |

## 合并顺序

1. 先提交 P0 汇总，避免后续结果表继续漏项。

4. 三组结果齐后，再统一更新最终结论和推荐配置。
